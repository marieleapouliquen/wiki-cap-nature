Forum des enseignants & éducateurs

Un espace pour échanger entre praticiens : retours d'expérience, questions, partage de bonnes idées.

Bienvenue sur le forum

Ce forum est ouvert à toutes les personnes qui animent (ou souhaitent animer) un atelier Cap Nature : enseignant·e·s, animateur·ice·s nature, éducateur·ice·s à l'environnement, médiateur·ice·s scientifiques. Que vous prépariez votre premier atelier ou que vous en ayez déjà animé dix, votre voix compte.

ℹ️ Comment ça marche

Le forum est hébergé sur GitHub Discussions et intégré ici via l'outil libre giscus. Vous pouvez :

  • Lire toutes les discussions sans compte (aucune inscription nécessaire pour parcourir)
  • Poster, commenter, réagir avec un compte GitHub (gratuit, création en 30 secondes)
  • Recevoir des notifications par e-mail pour suivre une discussion
  • Utiliser le formatage Markdown (gras, listes, liens, images, code)

Discussions récentes

Toutes les discussions du forum apparaissent ci-dessous. Vous pouvez les parcourir librement, et participer si vous avez un compte GitHub.

💬 Le forum n'est pas encore configuré

Pour activer les discussions, l'administrateur·ice du Wiki doit :

  1. Activer les Discussions dans les paramètres du dépôt GitHub
  2. Installer l'application giscus sur le dépôt
  3. Générer la configuration sur giscus.app
  4. Remplacer le bloc <script src="https://giscus.app/client.js">...</script> dans forum/index.html

Voir la section configuration ci-dessous pour les instructions détaillées.

Comment participer

1. Créer un compte GitHub (si besoin)

Pour poster ou répondre, il faut un compte GitHub (gratuit, 30 secondes). C'est la même infrastructure que celle utilisée par des millions de développeurs et de scientifiques dans le monde.

  1. Aller sur github.com/signup
  2. Renseigner un e-mail, un mot de passe, un nom d'utilisateur (libre choix)
  3. Confirmer son e-mail
  4. Revenir sur cette page et participer !

💡 Conseil pour les enseignants

Utilisez un nom d'utilisateur qui reflète votre identité professionnelle (ex. marie-svt-rennes) plutôt qu'un pseudo opaque. Cela facilite les échanges entre pairs et donne du contexte à vos messages.

2. Lancer une nouvelle discussion

  • Choisir la catégorie appropriée (cf. plus haut)
  • Donner un titre clair (“Espèce non identifiée par Merlin sur platanes” plutôt que “Question”)
  • Décrire le contexte : niveau de classe, lieu, saison, problème rencontré
  • Ajouter une image si pertinent (capture d'écran de Merlin, photo de l'oiseau, sonogramme) — glisser-déposer dans la zone de saisie

3. Répondre, réagir, suivre

  • Cliquer sur Reply sous un message pour répondre dans le fil
  • Utiliser les réactions emoji (👍 ❤️ 🎉) pour valider sans alourdir le fil
  • Cliquer sur Subscribe pour recevoir un e-mail quand le fil s'enrichit
  • Si une réponse résout votre question : marquer comme Answer — c'est utile pour les futurs lecteurs

Charte du forum

🤝 Quelques règles pour que ça se passe bien

  • Bienveillance avant tout. Beaucoup de participants débutent ; chacun pose ses questions sans crainte du jugement.
  • Pas de question bête. Si vous vous demandez, d'autres aussi. Posez.
  • Restez factuel·le. Les retours négatifs sont bienvenus s'ils décrivent une situation, pas une personne.
  • Respect du RGPD. Ne publiez pas de photos identifiantes d'élèves sans autorisation. Anonymisez les copies/questionnaires.
  • Ne partagez pas d'observations sensibles. Pour les espèces protégées rares (chouette, rapaces nicheurs), ne précisez pas la localisation exacte — préférez la commune ou le quartier.
  • Citez vos sources. Si vous reprenez une information lue ailleurs, indiquez d'où elle vient.
  • Pas de publicité. Le forum n'est pas un canal commercial. Partager une formation gratuite ou un événement public est en revanche bienvenu (catégorie Annonces).

Les contenus du forum sont publiés sous la même licence CC BY-SA 4.0 que le reste du Wiki : ils peuvent être cités et réutilisés, en mentionnant les auteur·ice·s.

Configuration du forum (admin)

Cette section concerne uniquement la personne qui administre le dépôt GitHub. Une fois ces étapes effectuées, le forum sera pleinement fonctionnel.

Étape 1 · Activer GitHub Discussions sur le dépôt

  1. Aller dans Settings du dépôt
  2. Dans la section Features, cocher Discussions
  3. Aller dans l'onglet Discussions du dépôt, cliquer sur Set up discussions

Étape 2 · Créer les catégories

Dans Discussions → Categories, créer ces catégories (en cliquant sur le crayon ✏️) :

NomTypeDescriptionEmoji
Premiers pasDiscussionPremier atelier, doutes de débutant🌱
Retours d'expérienceDiscussionCe qui a marché ou pas📝
Matériel & applicationsQ&AQuestions techniques (le format Q&A permet de marquer les réponses validées)🎙️
Adaptations pédagogiquesDiscussionAdaptations par niveau, discipline, public🔄
Identifications & observationsDiscussionPartage d'observations, identifications surprenantes🐦
Prolongements en classeDiscussionIdées et productions d'élèves📚
Suggestions & améliorationsIdeaPour faire évoluer le Wiki (format Idea = avec votes)💡
Annonces & événementsAnnouncementRéservé aux modérateurs📣

Étape 3 · Installer giscus

  1. Aller sur github.com/apps/giscus
  2. Cliquer sur Install
  3. Choisir Only select repositories et sélectionner le dépôt du Wiki

Étape 4 · Générer la configuration giscus

  1. Aller sur giscus.app
  2. Renseigner le nom du dépôt (ex. marie-pouliquen/wiki-cap-nature)
  3. Choisir la catégorie de mappage : General (ou une catégorie dédiée “Forum”)
  4. Choisir l'option Discussion title contains page pathname
  5. Le site génère un bloc <script> à copier

Étape 5 · Coller le script dans cette page

  1. Ouvrir docs/forum/index.html
  2. Repérer le bloc BLOC GISCUS À DÉCOMMENTER ET CONFIGURER (commentaire HTML)
  3. Supprimer le placeholder <div class="giscus-placeholder">...</div>
  4. Décommenter le bloc giscus et y coller les valeurs data-repo, data-repo-id, data-category, data-category-id générées à l'étape 4
  5. Commit, push, et le forum est en ligne 🎉

Étape 6 · Mettre à jour les liens des catégories

Une fois les catégories créées, remplacer les liens https://github.com/ dans les cartes ci-dessus par les URL réelles de vos catégories (du type https://github.com/UTILISATEUR/DEPOT/discussions/categories/premiers-pas).

⚠️ Modération

Pensez à désigner au moins un·e modérateur·ice en plus de l'administrateur·ice principal·e (via Settings → Collaborators). La modération consiste à : épingler les bonnes discussions, marquer les réponses validées, supprimer les messages hors charte, accueillir les nouveaux. Comptez environ 1 h par semaine pour un forum modérément actif.

Bonus · Personnaliser le thème

Le widget giscus accepte plusieurs thèmes. Pour rester cohérent avec le Wiki, utilisez data-theme="light" (par défaut) ou créez un thème custom avec les couleurs de la charte (bleu marine #1f2a6b, vert #94c11f, jaune #f7d038). Voir la doc giscus sur les thèmes custom.
